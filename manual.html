Coveo Security Checker

The purpose of the Coveo Security Checker is to give developers/clients/partners the ability to quickly check and validate the security of the content and the used API keys. The extension is only available for Coveo people.

The following will be checked on the Organization:

  • * Sources
    • * Are they secured
  • * Query Pipelines
    • * Is a Search Hub condition set
    • * Is there a filters override defined
  • * API Keys (only one of * should be true)
    • * Defined for Queries only*
    • * Defined for Admin (more than 50 priviliges)*
    • * Defined for Impersonate only*
    • * Defined for View All

The following will be checked on the Search Interface:

  • * Set debug=true
  • * Report pipeline used
  • * Report original set of results
  • * Report API Key or SearchToken used
  • * Show the priviliges of the API Key/Search Token (Queries only, Admin only, Impersonate only)
  • * Report original set of results with `@uri`: ORIGIN
  • * Set `pipeline` to `empty`
  • * Execute query again: set of results must be the same as `ORIGIN`
  • * Add 'Spoofing' Queries:
    • * `dq=@uri`
    • * `dq=@sfisvisibleinapp==true @sfisvisibleinpkb==false @sfisvisibleinprm==false @sfisvisibleincsp==false`
    • * Execute query again: set of results must be the same as `ORIGIN`

Usage

>

The extension must be used in this order:

  • * Log in into the Coveo Platform
  • * Then you can execute the `Check Organization`
    This will provide you with the details of the organization
  • * Next goto the Front-end search interface and activate `Search Tracker`
  • * Execute a couple of searches
  • * When ready, hit `Analyze Queries`
  • * View/Download the report
See also the **HELP** inside the Popup window of the Extension.
